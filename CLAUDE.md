# devops-tools

A collection of personal DevOps scripts. Each tool lives in its own subdirectory and is self-contained.


## Project structure

```
alias-advisor/        # Shell history analyzer that suggests aliases via TUI
  alias-advisor.sh    # Entry point — builds/runs Docker container
  docker/             # Dockerfile and entrypoint.sh for the container
  src/                # Python: analyzer.py (scoring), tui.py (curses TUI)

forklift/             # Docker Compose service migration tool
  forklift-cli.sh     # Single-file bash script — build/deploy commands
```

## Tools overview

### alias-advisor
Analyzes bash/zsh/fish history to suggest aliases ranked by frequency + recency. Runs entirely inside Docker (no host deps beyond Docker). Entry point: `./alias-advisor/alias-advisor.sh`.

### forklift-cli
Packages a running Docker Compose service (images + compose folder + encrypted volumes) into a timestamped archive for migration to another host. Requires Docker with the compose plugin. Entry point: `bash forklift/forklift-cli.sh`.

## Key conventions

- Tools are Docker-based — host dependencies are kept minimal (Docker only, or Docker + bash).
- Scripts are bash; internal logic that needs heavier tooling (`jq`, `yq`, `7z`, Python) runs inside containers.
- No build system or package manager at the repo root — each tool is standalone.
- Passwords/secrets: `vol-password` for forklift should be passed via env var (`VOL_PW`), not as a positional argument, to avoid exposure in `ps aux`.
